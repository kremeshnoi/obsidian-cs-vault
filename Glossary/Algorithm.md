# Algorithm

## Definition

A finite, well-defined sequence of steps that takes an input, processes it, and produces an output to solve a specific problem.

## Key Properties

- **Finiteness** — must terminate after a finite number of steps
- **Definiteness** — each step is precisely and unambiguously defined
- **Input/Output** — has zero or more inputs and one or more outputs
- **Effectiveness** — every step is basic enough to be carried out in finite time

## Not to Be Confused With

[[Pattern]] — a pattern describes a reusable structural approach to organizing code, while an algorithm is a concrete step-by-step procedure that can be formally analyzed for correctness and complexity.

> [!note]
> Unlike patterns, algorithms have provable properties — time complexity, space complexity, correctness guarantees. This is what makes them formally analyzable and comparable.
